# Release Notes — v1.3.0

**Release Version:** 1.3.0
**Release Date:** 2026-07-16
**Baseline:** Sprint 11 – Version 1.3

## Executive Summary

AI-Benchmark v1.3.0 is a quality-only, additive sprint that improves type safety, test coverage, resource lifecycle hygiene, and CI enforcement on the frozen architecture baseline AD-61 through AD-75. All changes are backward compatible and fully covered by regression tests.

## Release Overview

Sprint 11 focuses on engineering quality improvements:
- Complete MyPy strict-mode cleanup across all production files.
- Test coverage expansion from 93.52% to 95.03%.
- Elimination of project-intrinsic SQLite ResourceWarnings.
- CI coverage threshold enforcement (fail_under = 95).
- 54 new tests covering engine fallback paths, analytics edge cases, and cross-cutting gaps.
- Complete governance evidence pack committed and synchronized to origin.

## Major Improvements

### Type Safety
- Resolved all MyPy strict-mode errors (35 → 0) across 70 source files.
- Added explicit type annotations in `evaluation/__init__.py`, `history.py`, `analytics.py`, `config.py`, `model_selector.py`, `plugin/registry.py`, `plugin/manager.py`.
- Used `typing.cast()` in plugin reporters to satisfy strict typing without altering runtime behavior.
- Added defensive `isinstance` checks in `config.py` to handle edge cases safely.

### Test Coverage
- Expanded coverage from 93.52% to 95.03%.
- Added `test_sprint11_coverage_engine.py` (19 tests) covering BenchEngine fallback strategies, circuit breaker, retry/timeout loops, provider initialization errors, and exception handling.
- Added `test_sprint11_coverage_analytics.py` (9 tests) covering reliability scoring, latency parsing, recommendations, trend building, and best-value selection.
- Added `test_sprint11_coverage_gaps.py` (26 tests) covering history lifecycle, model selector strategies, config validation, and edge cases.
- CI now enforces 95% coverage threshold via `[tool.coverage.report] fail_under = 95`.

### Resource Lifecycle
- Fixed SQLite connection lifecycle in `history.py`: wrapped `load_latest()` in `try/finally` with explicit `conn.close()`.
- Removed ownership guards in `recent_category_performance()` and `recent_runs()` to unconditionally close connections.
- Removed `filterwarnings = ["ignore::ResourceWarning"]` suppression from `pyproject.toml`.
- Updated test fixtures with missing `conn.close()` and `HistoryWriter.reset()`.

### Developer Tooling
- 54 new tests with 100% local self-coverage on each new file.
- Regression suite increased from 479 to 495 passing tests.
- No external dependencies added.

### Governance
- Committed all Sprint 11 lifecycle documents.
- Synchronized repository to origin.
- Created annotated tag `v1.3.0` for immutable engineering baseline.
