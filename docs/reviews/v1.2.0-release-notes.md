# Release Notes — v1.2.0

**Release Version:** 1.2.0

## Executive Summary

AI-Benchmark v1.2.0 is a quality-only, additive sprint that improves type safety, lint hygiene, test coverage, and developer documentation without altering architecture, behavior, or public interfaces. All changes are backward compatible and fully covered by regression tests.

## Release Overview

Sprint 10 focuses on code quality and documentation hygiene:
- Comprehensive type annotation coverage across engine boundaries, CLI entry points, plugins, and test files.
- Elimination of all Ruff lint findings.
- Expansion of regression test suite with meaningful production-path coverage.
- Complete developer documentation and governance evidence pack.

## Major Sprint 10 Improvements

### Type Safety
- Added return-type and parameter-type annotations to `cli.py` Click commands.
- Added type annotations to all benchmark plugin `run()` signatures.
- Added type annotations to all provider plugin `__init__()` and `chat()` signatures.
- Added type annotations to reporter plugin `generate()` kwargs.
- Extracted local variables in `model_selector.py` and `engine.py` to satisfy mypy without changing runtime logic.
- Replaced bare `Any` with typed hints under `TYPE_CHECKING` in `prompts.py`.

### Lint Hygiene
- Removed unused imports in `memory_profiler.py`, `validation.py`, `rc_validation.py`, `parallel_executor.py`.
- Replaced unused imports with `typing.TYPE_CHECKING` in `prompts.py`.
- Result: `ruff check aibenchmark/` exits 0.

### Test Coverage
- Added 48 new tests across 6 files covering:
  - Auth / credential resolution edge cases
  - Auto-validation error paths (empty results, missing metadata, outliers)
  - Config validation (routing, fallback, parallel, cost, capability score)
  - ExecutionPolicy circuit-breaker transitions
  - PluginManager error handling
  - Validation helpers (path traversal, schema constraints)
- Coverage: 93.52% raw → 94% reported under Sprint 10 rounding policy.

### Governance Documentation
- Established complete Sprint 10 governance evidence pack:
  - Planning, Technical Debt Register, Implementation, Internal QA, QA Resolution, QA Re-Validation, RC Validation, RC Validation Resolution, RC Re-Validation, Acceptance Review, Formal Acceptance, Repository Audit, Repository Audit Resolution, Repository Re-Audit, Release Snapshot.
- All documents committed to default branch and traceable via `scripts/validate_governance_docs.py`.

### Developer Experience
- `docs/developer-guide.md` covers setup, testing, coverage, plugin SDK, CI workflow, governance, contribution guidelines.
- README updated with developer guide link and accurate architecture/CLI sections.
